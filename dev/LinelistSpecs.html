LinelistSpecs

Modern facade that wraps a specifications workbook and exposes lazy-loaded, cached accessors for every collaborator object (dictionary, choices, translations, geo, analysis, passwords, format, exports, formulas, main). Provides lifecycle orchestration methods for importing data from external setup workbooks, preparing the dictionary with preserved sheet names, exporting specifications to a destination linelist workbook, and surfacing user-facing error dialogs. Uses a PredeclaredId factory pattern so callers construct instances via LinelistSpecs.Create(workbook) and receive an ILinelistSpecs interface reference.

Depends on: ILLdictionary, ILLChoices, ILLTranslation, IDesignerTranslation, ILLGeo, ILLExport, ILLAnalysis, IPasswords, ILLFormat, IFormulaData, IMain, ILLVariables, IFormulas, ITranslationObject, IApplicationState, ApplicationState, BetterArray, LLdictionary, LLChoices, LLTranslation, DesignerTranslation, LLGeo, LLExport, Analysis, Passwords, LLFormat, FormulaData, Main, LLVariables, Formulas

Workbook orchestration

prepare

Import and prepare all specifications from external workbooks

Signature:

Public Sub Prepare()

Lifecycle methods for importing, preparing, exporting, and error-managing the specifications workbook. Orchestrates the full specification preparation workflow:

  1. Validates Main sheet readiness via CheckReadiness.
  2. Opens the setup workbook and validates its required sheets.
  3. Imports exports, dictionary, choices, translations, and analysis data.
  4. Optionally opens and imports geo data from a separate workbook.
  5. Prepares the dictionary with preserved sheet names and geo references.
  6. Adds the "list auto" column if not already present.
  7. Translates and sorts all collaborators using dictionary translations. Handles errors with cleanup of open workbooks and busy-state restoration.

Throws:

  • ProjectError.ElementNotFound When paths or sheets are missing.
  • ProjectError.ErrorUnexpectedState When an unexpected error occurs.
